# Task Intake Template

## Objective

What outcome should be true after this task?

## Scope

What is in scope and explicitly out of scope?

## Constraints

- compatibility constraints
- dependency constraints
- performance constraints
- security or compliance constraints

## Acceptance Criteria

List concrete conditions for completion.

## Relevant Files or Areas

List likely files, modules, or subsystems involved.

## Verification

Name the commands, tests, screenshots, or measurements required.

## Risks

What could break if the change is done incorrectly?

## Notes

Any domain-specific context the agent should not have to guess.
