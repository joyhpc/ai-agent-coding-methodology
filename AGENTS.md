# AGENTS.md

This repository is itself maintained as an agent-friendly repository.

## Mission

Keep this repository focused on practical methodology for coding agents.

Every change should improve one of these:

- clarity of rules
- reliability of execution
- quality of verification
- usefulness of retrospectives

## Repository Rules

1. Prefer small, composable documents over one giant handbook.
2. Do not add prompt collections without a workflow context.
3. Every playbook should include failure modes and verification steps.
4. Every retrospective should end with at least one rule, template, or workflow update candidate.
5. Prefer source-backed claims; distinguish sourced facts from synthesis.
6. When documenting a method, state scope, assumptions, failure modes, and acceptance criteria.
7. If a lesson repeats, promote it into a stable rule or template.

## Writing Conventions

- Use concise, operational language.
- Prefer checklists and decision rules over vague advice.
- Mark examples clearly as examples, not universal law.
- Keep tool-specific guidance separate from general methodology when possible.

## Change Policy

- New documents should have a clear owner domain: `docs/`, `playbooks/`, `templates/`, `research/`, or `retrospectives/`.
- Cross-reference rather than duplicating content.
- If a new concept changes the overall model, update `README.md` and `docs/METHODOLOGY.md`.
